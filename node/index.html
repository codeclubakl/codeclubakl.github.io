Node web app

This tutorial assumes you've already done the NodeStart activity, where you made a basic web app that lets you post ideas and see an ideas feed.

Now what?

Time to build your own web app. You can copy the code from the NodeStart activity as a starting point, then make changes.

If you know how to use Github, you can work together with some other people to build an app. This will help you make a lot more progress, because there is lots of work to be done.

Creating a new Github project

One person per team: go to github.com and log in to your account. Click on the New repository button to create a new repository, give it a name and create it.

And the rest of the team as collaborators. From the repository page, click settings, then collaborators, then add each team member as a collaborator.

Now everyone needs to go to the repository page and click 'Clone in Desktop' to clone the project onto their own computer, so they can work.

Making a change

Decide what you are going to work on. Look at your project's Issues page, if there are no issues create one to work on. Assign the issue to yourself so everyone knows it is being worked on, this stops two people trying to change the same thing at the same time.

In Github Desktop, make sure you are up to date (click 'Sync') then create a new Branch for your changes. Include the issue number in the branch name.

Work on your issue, making changes. Whenever you have one step complete and working, commit it using Github Desktop. Once you've finished, use Github Desktop to create a Pull Request. This means you are ready for your changes to be put into the main project.

Ideas to work on:

  1. On post.html, when you click submit, it should NOT take you to a new page. It should just say 'thanks for your idea'
  2. Put a link bar on every page so its easy to navigate between the pages
  3. On the server, save each ideas as a javascript object {} instead of just a string, so we can add metadata like the author's name, and the time it was posted
  4. (depends on #3) When you post an idea, you can type in your name (the 'author') and this is saved and displayed in the feed
  5. (depends on #3) When you post an idea, the server records the time it was posted. This is displayed in the feed.
  6. Make the post.html form look better
  7. Make the feed.html ideas look better
  8. Make a login page, even if it doesn't work yet
  9. Depends on 8: Make the login page give the user a cookie with their username. When they post, use the cookie to get their name instead of making them type it in every time.
  10. Publish the app on Heroku so anyone can use it. Set up Heroku to automatically update whenever someone updates the project.
